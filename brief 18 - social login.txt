## KONTEKST

FastHub ma kompletny framework OAuth 2.0 z PKCE (fasthub_core/integrations/oauth.py),
ale brakuje gotowych providerów social login. Użytkownik musi logować się emailem i hasłem.

Cel: dodać "Zaloguj się przez Google / GitHub / Microsoft" — klik i działa.

## CO JEST
- OAuthManager z PKCE flow (authorization_url, exchange_code, refresh_tokens)
- OAuthConfig dataclass (client_id, client_secret, scopes, urls)
- DatabaseTokenStorage z Fernet encryption
- Token refresh logic
- Config: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET (tylko Google)

## CZEGO BRAKUJE
1. User model — pola social ID (google_id, github_id, microsoft_id)
2. Provider configs — gotowe konfiguracje Google/GitHub/Microsoft z URL-ami
3. Backend endpoints — /auth/{provider}/login → redirect, /auth/{provider}/callback → JWT
4. Logika user linking — OAuth email → istniejący user? link : nowy user
5. Frontend — przyciski social login na stronie logowania
6. Config — zmienne dla GitHub i Microsoft
7. Migracja Alembic — nowe kolumny w users

## CZEGO NIE ROBIĆ
- NIE usuwać email+password login — social login jest DODATKIEM
- NIE wymagać social login — opcjonalne
- NIE budować own OAuth server — tylko client (logowanie PRZEZ Google, nie login DO FastHub przez OAuth)
