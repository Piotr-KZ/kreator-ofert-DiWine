rief 18 — Data Export (GDPR) — wersja rozszerzona:

Kontekst prawny
RODO (GDPR) wymaga trzech rzeczy od każdego SaaS:
Prawo dostępu (Art. 15) — użytkownik pyta "jakie moje dane macie?" → musisz dać mu pełny eksport w czytelnym formacie (JSON, CSV, PDF).
Prawo do usunięcia (Art. 17) — użytkownik mówi "usuńcie wszystko o mnie" → musisz usunąć dane osobowe. ALE: faktury i dane księgowe musisz trzymać 5 lat (ustawa o rachunkowości). Audit trail też powinien zostać (ale zanonimizowany).
Prawo do przenoszenia (Art. 20) — dane w formacie maszynowym (JSON) żeby mógł je zabrać do konkurencji.

### B) Anonymize Service — "Usuń moje konto"

NIE kasujemy rekordów. Zamieniamy dane osobowe na hash:
```
PRZED:
    User: email="jan@firma.pl", full_name="Jan Kowalski"
    Audit: user_email="jan@firma.pl", ip_address="192.168.1.1"
    Organization: owner_id=UUID(...)

PO ANONIMIZACJI:
    User: email="deleted_a1b2c3@anonymized.local", full_name="Deleted User a1b2c3"
    User: is_active=False, is_verified=False, hashed_password="DELETED"
    Audit: user_email="deleted_a1b2c3@anonymized.local", ip_address="0.0.0.0"
    Organization: owner_id=NULL (przeniesiony na innego admina lub usunięty)
```

Dlaczego nie kasowanie:
- Faktury — prawo wymaga 5 lat przechowywania. Ale dane osobowe na fakturze zamieniamy (nabywca = "Deleted User")
- Audit trail — potrzebny do bezpieczeństwa. Ale email i IP zamazane
- Subskrypcje — historia płatności zostaje (bez danych osobowych)
- Organizacja — jeśli user był jedynym właścicielem, organizacja się dezaktywuje. Jeśli byli inni członkowie, ownership przechodzi na następnego admina

### C) Deletion Request — workflow usunięcia

Użytkownik nie dostaje natychmiastowego usunięcia. Jest grace period:

1. User klika "Usuń konto" → DeletionRequest (status=pending, requested_at, execute_at=+14 dni)
2. System wysyła email "Czy na pewno? Masz 14 dni na anulowanie"
3. Po 14 dniach → cron job (task queue z Brief 15) uruchamia anonimizację
4. Przed anonimizacją → automatyczny eksport danych (wysłany mailem jako ZIP)
5. Anonimizacja → status=completed

Dlaczego 14 dni: RODO mówi "bez zbędnej zwłoki, nie później niż miesiąc". 14 dni to standard branżowy — chroni przed przypadkowym usunięciem i daje czas na "rozmyśliłem się".

from fasthub_core.gdpr import ExportRegistry, DataExporter

class ProcessExporter(DataExporter):
    async def export_user_data(self, user_id, db):
        processes = await db.execute(
            select(Process).where(Process.created_by == user_id)
        )
        return {"processes": [p.to_dict() for p in processes.scalars()]}
    
    async def get_export_name(self):
        return "processes"

ExportRegistry.register(ProcessExporter())
