## KONTEKST

Brief 10 przeniósł 7 uniwersalnych modułów z AutoFlow do `fasthub_core/` w repo FastHub. Teraz AutoFlow ma **duplikaty** — lokalne kopie modułów PLUS te same moduły w fasthub_core.

Ten brief eliminuje duplikaty: AutoFlow przestaje używać lokalnych `app.core.redis`, `app.core.encryption`, `app.events`, itd. i zamiast tego importuje z `fasthub_core`.

**Skala pracy:**
- ~47 importów do zamiany w ~15 plikach

- 1 plik do modyfikacji (main.py — startup sequence)
- 2 pliki testowe do aktualizacji

from fasthub_core.events.bus import Event, EventBus, event_bus  # noqa: F401

# =============================================================================
# Te handlery SĄ specyficzne dla AutoFlow (obsługa procesów, integracji)

async def _log_execution_failure(event):
    """Loguj błędy wykonania procesów."""
    logger.error(f"Execution failed: {event.get('payload', {}).get('error', 'unknown')}")

async def _log_health_warning(event):
    """Loguj ostrzeżenia o zdrowiu integracji."""
    logger.warning(f"Integration health: {event.get('payload', {})}")

async def _log_system_events(event):
    """Loguj eventy systemowe."""

## KROK 6: ENV VARS

### 6A: Sprawdź co Brief 10 zrobił z encryption key

Brief 10 miał zmienić `AUTOFLOW_SECRET_KEY` → `FASTHUB_SECRET_KEY`. fasthub_core powinien czytać `FASTHUB_SECRET_KEY`.

Ale AutoFlow w produkcji ma `AUTOFLOW_SECRET_KEY` w `.env`. Potrzebujemy **fallback**:

Sprawdź czy `fasthub_core/security/encryption.py` ma fallback:
```python
SECRET_KEY = os.environ.get('FASTHUB_SECRET_KEY') or os.environ.get('AUTOFLOW_SECRET_KEY')
```

**Jeśli NIE ma** — dodaj do thin wrappera `app/core/encryption.py`:
```python
import os
# Backward compat: ustaw FASTHUB_SECRET_KEY jeśli tylko AUTOFLOW_SECRET_KEY istnieje
if not os.environ.get('FASTHUB_SECRET_KEY') and os.environ.get('AUTOFLOW_SECRET_KEY'):
    os.environ['FASTHUB_SECRET_KEY'] = os.environ['AUTOFLOW_SECRET_KEY']

from fasthub_core.security.encryption import (  # noqa: F401
    encrypt_credentials, decrypt_credentials,
    rotate_encryption_key, generate_key,
    mask_credentials, is_encryption_available,
)
```

        assert len(ALL_PERMISSIONS) >= 20  # 22 w oryginalnym manifeście

    def test_manifest_covers_code_permissions(self):
        """Sprawdź że manifest pokrywa permissions z kodu."""
        import subprocess, os
        root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
        result = subprocess.run(
            ["grep", "-rn", 'require_permission("', "--include=*.py", "app/"],
            capture_output=True, text=True, cwd=root
        )
        from app.fasthub_manifest import ALL_PERMISSIONS
        code_perms = set()
        for line in result.stdout.strip().split("\n"):
            if 'require_permission("' in line:
                perm = line.split('require_permission("')[1].split('"')[0]
                code_perms.add(perm)

        missing = code_perms - set(ALL_PERMISSIONS)
        assert not missing, f"Permissions w kodzie ale nie w manifeście: {missing}"
