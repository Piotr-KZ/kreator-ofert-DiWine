### Krok 2: Stwórz plik `fasthub_core/contracts.py`

To jest serce kontraktu. Plik definiuje INTERFEJSY (abstrakcyjne klasy) — mówi CO FastHub dostarcza, bez mówienia JAK to robi wewnętrznie.

```python
"""
FastHub Core — Kontrakt API v2.0

Ten plik definiuje interfejsy które FastHub GWARANTUJE dostarczyć.
Aplikacje (AutoFlow, FastCRM, etc.) polegają na tych interfejsach.

ZASADY:
1. Nie zmieniaj sygnatur funkcji bez podniesienia MAJOR version (v3.0)
2. Nowe funkcje DODAWAJ — nie zmieniaj istniejących
3. Usuwanie funkcji — minimum 1 wersja z deprecation warning
"""

from abc import ABC, abstractmethod
from typing import List, Optional, Set, Dict, Any
from uuid import UUID
from datetime import datetime


class AuthContract(ABC):
    """
    Kontrakt autentykacji — logowanie, tokeny, hasła.
    AutoFlow potrzebuje tego żeby wiedzieć kto jest zalogowany.
    """
    
    @abstractmethod
    async def hash_password(self, password: str) -> str:
        """Zamienia hasło na bezpieczny hash (bcrypt)"""
        ...
    
    @abstractmethod
    async def verify_password(self, plain_password: str, hashed_password: str) -> bool:
        """Sprawdza czy podane hasło zgadza się z hashem"""
        ...
    
    @abstractmethod
    async def create_access_token(self, user_id: str, organization_id: Optional[str] = None, extra_data: Optional[Dict] = None) -> str:
        """Tworzy krótkotrwały token dostępu (JWT). Czas życia: ~60 min."""
        ...
    
    @abstractmethod
    async def create_refresh_token(self, user_id: str) -> str:
        """Tworzy długotrwały token odświeżania. Czas życia: ~30 dni."""
        ...
    
    @abstractmethod
    async def decode_token(self, token: str) -> Dict[str, Any]:
        """Dekoduje token JWT. Zwraca payload z user_id, exp, type."""
        ...
    
    @abstractmethod
    async def blacklist_token(self, token_jti: str, ttl_seconds: int) -> None:
        """Unieważnia token (dodaje do czarnej listy w Redis)"""
        ...
    
    @abstractmethod
    async def is_token_blacklisted(self, token_jti: str) -> bool:
        """Sprawdza czy token jest na czarnej liście"""
        ...


class UserContract(ABC):
    """
    Kontrakt użytkowników — informacje o ludziach i firmach.
    AutoFlow potrzebuje tego do: dropdown akceptacji, identyfikacja usera.
    """
    
    @abstractmethod
    async def get_current_user(self, token: str) -> Dict[str, Any]:
        """
        Zwraca dane zalogowanego użytkownika na podstawie tokenu.
        Zwraca: {id, email, full_name, is_active, organization_id}
        """
        ...
    
    @abstractmethod
    async def get_user(self, user_id: str) -> Optional[Dict[str, Any]]:
        """Zwraca dane użytkownika po ID"""
        ...
    
    @abstractmethod
    async def list_organization_users(self, organization_id: str) -> List[Dict[str, Any]]:
        """
        Lista wszystkich użytkowników w organizacji.
        AutoFlow używa do: dropdown "kto ma zatwierdzić proces"
        Zwraca: [{id, email, full_name, role}, ...]
        """
        ...
    
    @abstractmethod
    async def get_user_role(self, user_id: str, organization_id: str) -> Optional[str]:
        """Zwraca rolę użytkownika w organizacji (np. 'admin', 'member')"""
        ...


class PermissionContract(ABC):
    """
    Kontrakt uprawnień — kto co może robić.
    AutoFlow potrzebuje: sprawdzanie czy user może edytować/wykonać proces.
    UWAGA: Ten kontrakt będzie ROZSZERZONY w FastHub v2.0 (Advanced RBAC).
    """
    
    @abstractmethod
    async def check_permission(self, user_id: str, organization_id: str, permission: str) -> bool:
        """
        Sprawdza czy user ma dane uprawnienie w organizacji.
        Przykłady permission: "processes.edit", "processes.execute", "billing.view"
        """
        ...
    
    @abstractmethod
    async def get_user_permissions(self, user_id: str, organization_id: str) -> Set[str]:
        """Zwraca zestaw wszystkich uprawnień usera w organizacji"""
        ...


class BillingContract(ABC):
    """
    Kontrakt rozliczeń — plany, limity, zużycie.
    AutoFlow potrzebuje: sprawdzanie limitów (ile procesów może mieć firma).
    """
    
    @abstractmethod
    async def get_subscription(self, organization_id: str) -> Optional[Dict[str, Any]]:
        """
        Zwraca aktualną subskrypcję organizacji.
        Zwraca: {plan_name, status, limits: {processes: 50, ai_calls: 1000, ...}}
        """
        ...
    
    @abstractmethod
    async def check_limit(self, organization_id: str, resource: str, current_usage: int) -> bool:
        """
        Sprawdza czy organizacja nie przekroczyła limitu.
        resource: "processes", "ai_calls", "integrations", "members"
        Zwraca: True jeśli OK (nie przekroczono), False jeśli limit osiągnięty
        """
        ...
    
    @abstractmethod
    async def record_usage(self, organization_id: str, resource: str, amount: int = 1) -> None:
        """
        Zapisuje zużycie zasobu (np. +1 wywołanie AI, +1 wykonanie procesu).
        Używane do mierzenia zużycia i egzekwowania limitów.
        """
        ...


class AuditContract(ABC):
    """
    Kontrakt audytu — historia zmian.
    AutoFlow potrzebuje: logowanie zmian w procesach.
    """
    
    @abstractmethod
    async def log_action(
        self,
        user_id: str,
        organization_id: str,
        action: str,
        resource_type: str,
        resource_id: str,
        details: Optional[Dict[str, Any]] = None,
        before: Optional[Dict[str, Any]] = None,
        after: Optional[Dict[str, Any]] = None,
    ) -> None:
        """
        Zapisuje akcję w logu audytu.
        action: "create", "update", "delete", "execute", "approve", "reject"
        resource_type: "process", "integration", "user", "subscription"
        before/after: stan zasobu PRZED i PO zmianie (opcjonalne)
        """
        ...
    
    @abstractmethod
    async def get_audit_logs(
        self,
        organization_id: str,
        resource_type: Optional[str] = None,
        resource_id: Optional[str] = None,
        user_id: Optional[str] = None,
        limit: int = 50,
        offset: int = 0,
    ) -> List[Dict[str, Any]]:
        """Pobiera logi audytu z filtrami"""
        ...


class NotificationContract(ABC):
    """
    Kontrakt powiadomień — wysyłanie powiadomień do użytkowników.
    AutoFlow potrzebuje: informowanie o zakończeniu procesu, oczekującej akceptacji.
    UWAGA: Ten kontrakt będzie DODANY w FastHub v2.0 (jeszcze nie zaimplementowany).
    """
    
    @abstractmethod
    async def send_notification(
        self,
        user_id: str,
        notification_type: str,
        title: str,
        message: str,
        data: Optional[Dict[str, Any]] = None,
    ) -> None:
        """
        Wysyła powiadomienie do użytkownika (in-app + opcjonalnie email).
        notification_type: "info", "success", "warning", "error", "approval_request"
        """
        ...
    
    @abstractmethod
    async def send_email(
        self,
        to_email: str,
        template: str,
        variables: Dict[str, Any],
    ) -> None:
        """
        Wysyła email na podstawie szablonu.
        template: "welcome", "password_reset", "approval_request", "execution_complete"
        """
        ...


class DatabaseContract(ABC):
    """
    Kontrakt bazy danych — sesja, engine.
    Każda aplikacja potrzebuje dostępu do bazy danych.
    """
    
    @abstractmethod
    async def get_db_session(self):
        """
        Zwraca async session do bazy danych.
        Używane jako FastAPI dependency: Depends(get_db)
        """
        ...
    
    @abstractmethod
    def get_engine(self):
        """Zwraca SQLAlchemy async engine"""
        ...
```

### Krok 3: Stwórz plik `fasthub_core/contracts_impl.py`

Ten plik ŁĄCZY kontrakty z faktycznym kodem który już mamy w `fasthub_core/`. Mapuje abstrakcyjne interfejsy na realne implementacje.

```python
"""
Implementacja kontraktów — mapowanie interfejsów na realne funkcje.
Aplikacje importują te implementacje, nie interfejsy.
"""

from fasthub_core.contracts import (
    AuthContract, UserContract, PermissionContract,
    BillingContract, AuditContract, NotificationContract, DatabaseContract
)

# Importy faktycznych implementacji z fasthub_core/
from fasthub_core.auth.service import (
    get_password_hash, verify_password,
    create_access_token, create_refresh_token, decode_token
)
from fasthub_core.auth.token_blacklist import TokenBlacklist
from fasthub_core.db.session import get_db, engine
from fasthub_core.audit.service import AuditService

# ... itd. — dopasuj importy do faktycznych funkcji w fasthub_core/


class FastHubAuth(AuthContract):
    """Implementacja kontraktu auth oparta na fasthub_core.auth"""
    
    async def hash_password(self, password: str) -> str:
        return get_password_hash(password)
    
    async def verify_password(self, plain_password: str, hashed_password: str) -> bool:
        return verify_password(plain_password, hashed_password)
    
    # ... implementuj pozostałe metody mapując na faktyczne funkcje


# UWAGA: Nie wszystkie kontrakty mają jeszcze implementację!
# NotificationContract — będzie w FastHub v2.0
# PermissionContract — na razie basic (admin/member), rozbudowa w v2.0
```

**WAŻNE:** Nie wymyślaj implementacji których nie ma. Jeśli `fasthub_core/` nie ma jeszcze jakiejś funkcji — zostaw `raise NotImplementedError("Planowane w FastHub v2.0")`. To normalne — kontrakt mówi co BĘDZIE, implementacja nadchodzi stopniowo.

# FastHub Core — Kontrakt API v2.0

## Dla kogo ten dokument
Dla programistów budujących aplikacje na FastHub (np. AutoFlow, FastCRM).

## Zasady kontraktu
1. Sygnatury funkcji nie zmieniają się w ramach tej samej MAJOR wersji
2. Nowe funkcje są DODAWANE — stare nie znikają
3. Deprecated funkcje działają minimum 1 wersję po ogłoszeniu
4. Breaking changes tylko w nowej MAJOR wersji (v3.0)

## Dostępne kontrakty

### AuthContract — Logowanie i tokeny
| Metoda | Status | Opis |
|--------|--------|------|
| hash_password(password) | ✅ Gotowe | Hashowanie hasła bcrypt |
| verify_password(plain, hashed) | ✅ Gotowe | Weryfikacja hasła |
| create_access_token(user_id, org_id) | ✅ Gotowe | JWT access token (~60min) |
| create_refresh_token(user_id) | ✅ Gotowe | JWT refresh token (~30d) |
| decode_token(token) | ✅ Gotowe | Dekodowanie JWT |
| blacklist_token(jti, ttl) | ✅ Gotowe | Unieważnienie tokenu |
| is_token_blacklisted(jti) | ✅ Gotowe | Sprawdzenie czarnej listy |

### UserContract — Użytkownicy i organizacje
| Metoda | Status | Opis |

| get_current_user(token) | ✅ Gotowe | Zalogowany user z tokenu |

| list_organization_users(org_id) | ✅ Gotowe | Wszyscy userzy w firmie |
| get_user_role(user_id, org_id) | ✅ Gotowe | Rola usera w firmie |

### PermissionContract — Uprawnienia
| Metoda | Status | Opis |
|--------|--------|------|

### BillingContract — Rozliczenia

|--------|--------|------|
