## KONTEKST

AutoFlow v7 ma 7 modułów które są **uniwersalne** — każdy SaaS ich potrzebuje, nie tylko platforma automatyzacji procesów. Te moduły powinny żyć w `fasthub_core/` żeby inne aplikacje (AutoFlow, FastCRM, cokolwiek zbudowane na FastHub) mogły z nich korzystać.

**Dlaczego teraz?** Brief 3 (rename Workspace→Organization) właśnie się skończył — AutoFlow i FastHub mówią tym samym językiem. Poprawki FastHub (unified BaseModel, RBAC activation, cleanup blacklist) robią się równolegle. Ten brief dodaje fundamentalne serwisy infrastrukturalne do fasthub_core.

**Co przenosimy (7 modułów, ~3400 linii kodu źródłowego):**

|---|-------|-------------------|-------|---------|

| B | Encryption | `app/core/encryption.py` | 263 | Szyfrowanie credentials (API keys, OAuth tokens) — AES-128 Fernet |
| C | Redis Service | `app/core/redis.py` | 372 | Connection pool + cache + pub/sub + health check — fundament dla Event Bus, cache, queues |

### Co przenieść

**Connection Pool Singleton** — jedna pula połączeń Redis dla całej aplikacji:


4. **Kompatybilność z istniejącym blacklist** — FastHub już używa Redis w `auth/blacklist.py`. Nowy moduł musi współdzielić connection pool (nie tworzyć drugiego). Sprawdź jak blacklist tworzy połączenie i UJEDNOLIĆ:

## MODUŁ A: EVENT BUS (KROK 2)

### Źródło
`autoflow/app/events/__init__.py` — 86 linii (plus default handlers)

### Cel
`fasthub_core/events/bus.py`

### Co przenieść

**Klasa Event:**
```python
class Event:
    event_type: str    # np. "user.login", "subscription.created"
    payload: Dict      # dane eventu
    timestamp: str     # ISO 8601
    source: str        # "system", "api", "webhook"
```

**Klasa EventBus (singleton):**
```python
class EventBus:
    def on(self, event_pattern: str)           # Dekorator: @event_bus.on("user.*")
    def register(self, pattern: str, handler)  # Programmatic registration
    async def emit(self, event_type: str, payload: dict, source: str = "system") -> int
    async def emit_many(self, events: List[tuple]) -> int
    def get_stats(self) -> Dict[str, Any]
    def clear_handlers()
```

**Wildcard matching** — `fnmatch` pattern:
- `user.*` — łapie `user.login`, `user.logout`, `user.created`
- `*.error` — łapie `process.error`, `integration.error`
- `*` — łapie WSZYSTKO

**Redis Pub/Sub** — emit() oprócz local handlers publikuje event na Redis (dla multi-server):
```python
# W emit():
await publish_event(event_type, event.to_dict())  # z redis.py
```

### Co ZMIENIĆ

1. **Usuń default handlers specyficzne dla AutoFlow:**
```python
# TE USUWAMY (są w autoflow/app/events/__init__.py):
async def _log_execution_failure(event)     # ← AutoFlow-specific
async def _log_health_warning(event)        # ← AutoFlow-specific
async def _log_system_events(event)         # ← AutoFlow-specific
```

2. **Zamień na generyczne default handlers:**
```python
# Nowe — logowanie eventów (uniwersalne):
async def _default_log_handler(event):
    """Loguje wszystkie eventy na INFO level"""
    logger.info(f"Event: {event.get('event_type')} from {event.get('source')}")

# Rejestracja:
event_bus = EventBus()
event_bus.register("system.*", _default_log_handler)
```

3. **Zmień import Redis:**
```python
# Zamiast: from app.core.redis import publish_event
# Nowe: from fasthub_core.infrastructure.redis import publish_event
```

4. **Zmień typowanie eventów na generyczne:**
```python
# AutoFlow ma: process.*, execution.*, integration.*, document.*, user.*, system.*
# FastHub powinien definiować BAZOWE:
# user.*, organization.*, billing.*, audit.*, system.*
# Aplikacje (AutoFlow) dodają swoje (process.*, execution.*, connector.*)
```

## MODUŁ B: ENCRYPTION (KROK 3)

### Źródło
`autoflow/app/core/encryption.py` — 263 linii

### Cel
`fasthub_core/security/encryption.py`

### Co przenieść — CAŁY PLIK z adaptacjami

**Kluczowe funkcje:**
- `encrypt_credentials(credentials: Dict) -> str` — szyfruje dict na string z prefixem "ENC:"
- `decrypt_credentials(encrypted_data: str) -> Dict` — deszyfruje z powrotem
- `rotate_encryption_key(old_key, new_key, encrypted_data) -> str` — rotacja klucza
- `generate_key() -> str` — generuje nowy Fernet key
- `mask_credentials(credentials: Dict) -> Dict` — maskuje wartości do logów (API k→ `"API ***"`)
- `is_encryption_available() -> bool` — sprawdza czy klucz jest ustawiony

**Lazy initialization** — nie wymaga cryptography przy starcie (dev mode działa bez szyfrowania).

**Fallback** — jeśli `SECRET_KEY` nie ustawiony → `encrypt_credentials()` zapisuje plain JSON (z ostrzeżeniem w logach).

**CLI** — `python encryption.py generate-key` i `python encryption.py test`

### Co ZMIENIĆ

1. **Zmień env var:**
```python
# Zamiast: AUTOFLOW_SECRET_KEY
# Nowe: FASTHUB_SECRET_KEY (lub z config.py)
from fasthub_core.config import settings
SECRET_KEY = getattr(settings, 'SECRET_KEY', None) or os.environ.get('FASTHUB_SECRET_KEY')
```

2. **Zmień prefix w logach:**
```python
# Zamiast: logger = logging.getLogger("autoflow.encryption")
# Nowe: logger = logging.getLogger("fasthub.encryption")
```

3. **Zachowaj backward compatibility prefix "ENC:"** — AutoFlow już ma dane zaszyfrowane z tym prefixem. Nie zmieniamy formatu.

```python
    decrypted = decrypt_credentials(re_encrypted)
    assert decrypted["secret"] == "data"
```

### INSTRUKCJA

**Brief 9 już dostarczył SecurityHeadersMiddleware.** Twoim zadaniem jest PORÓWNANIE — czy Brief 9 pokrywa te same nagłówki co AutoFlow:

AutoFlow ustawia:
- `X-Frame-Options: DENY`
- `Content-Security-Policy` (z dozwolonym `wss:` i `ws:` dla WebSocket)
- `Strict-Transport-Security` (warunkowy — tylko jeśli HSTS_ENABLED)
- `X-Content-Type-Options: nosniff`

**Jeśli Brief 9 pokrywa wszystko** — nic nie rób, ten moduł jest GOTOWY.

**Jeśli czegoś brakuje** — dodaj brakujące nagłówki do istniejącego `SecurityHeadersMiddleware`. NIE twórz nowego pliku.

### Test

```python
def test_security_headers_comparison():
    """Sprawdź czy wszystkie nagłówki z AutoFlow są w FastHub"""
    from fasthub_core.middleware.security_headers import SecurityHeadersMiddleware
    from starlette.testclient import TestClient
    from starlette.applications import Starlette
    from starlette.responses import PlainTextResponse
    from starlette.routing import Route

    async def homepage(request):
        return PlainTextResponse("OK")

    app = Starlette(routes=[Route("/", homepage)])
    app.add_middleware(SecurityHeadersMiddleware)
    client = TestClient(app)
    response = client.get("/")

    # Wymagane nagłówki (z AutoFlow):
    assert response.headers.get("X-Content-Type-Options") == "nosniff"
    assert response.headers.get("X-Frame-Options") == "DENY"
    csp = response.headers.get("Content-Security-Policy", "")
    assert "wss:" in csp or "ws:" in csp  # WebSocket musi być dozwolony
```

```python
    # UWAGA: Nie twórz billing_plans na nowo jeśli fasthub-backend
    # już ma subscriptions i invoices. Sprawdź najpierw!
    # Jeśli istnieją — dodaj nowe tabele OBOK (nie usuwaj starych).
```

### Co zrobić z istniejącym FastHub Subscription/Invoice

FastHub z Briefu 0 ma prostsze modele `Subscription` i `Invoice` w billing. AutoFlow ma bardziej rozbudowane.

**Decyzja:**
1. **Zachowaj istniejące** `Subscription` i `Invoice` jeśli backend ich używa
2. **Dodaj nowe** tabele z AutoFlow (billing_plans, billing_addons, tenant_addons, usage_records, billing_events)
3. **Wzbogać istniejący Subscription** o pola z AutoFlow (plan_id FK, billing_interval, stripe IDs) — via migracja ALTER
4. **NIE usuwaj Invoice** — może być używany
